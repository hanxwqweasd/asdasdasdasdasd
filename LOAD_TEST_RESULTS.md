# Контрольные испытания V2

Дата прогона: 14 июля 2026 года. Стенд: один Node.js-процесс, PostgreSQL 17 и Redis на локальной машине. Это проверка корректности и отсутствия гонок; цифры не являются обещанием производительности конкретного тарифа Railway.

## HTTP / первичное заселение

| Профиль | Запросы | Успешно | Ошибки | RPS | p50 | p95 | p99 |
|---|---:|---:|---:|---:|---:|---:|---:|
| 100 пользователей × 2 запроса | 200 | 200 | 0 | 168 | 263 мс | 383 мс | 417 мс |
| 500 пользователей | 500 | 500 | 0 | 167 | 587 мс | 714 мс | 786 мс |
| 1 000 пользователей | 1 000 | 1 000 | 0 | 218 | 614 мс | 859 мс | 892 мс |

Проверено параллельное создание профилей и распределение по подъездам. После введения PostgreSQL advisory lock конфликтов кодов подъездов не осталось.

## Кооператив

- 25 матчей / 50 игроков: 25 завершённых сценариев проверки, 0 ошибок, 647 мс.
- 100 матчей / 200 игроков: 100 завершённых сценариев проверки, 0 ошибок, 1 785 мс.
- В каждом матче проверялись: создание, вход по коду, готовность, запуск, два синхронных действия и голосование.
- После остановки и перезапуска Node-процесса активная вылазка была восстановлена из Redis/PostgreSQL: фаза `playing`, улики и общее состояние сохранились.

## Данные и экономика

- Полностью пройдено девятишаговое обучение через HTTP API.
- Пройден ежедневный сценарий, выбрана профессия, сохранены аналитическое событие и обращение в поддержку.
- Рынок: escrow, покупка, комиссия 5%, передача предмета и provenance — успешно.
- Telegram Stars fulfillment был вызван дважды с одним charge payload: статус `paid`, начисление предмета ровно один раз, `stars_spent` увеличен ровно один раз.

## Backup

- Выполнен `pg_dump` custom-format.
- Копия восстановлена в отдельную пустую базу PostgreSQL.
- Проверено наличие таблиц `users`, `purchases`, `inventory` и чтение данных.
- Выполнен CSV-экспорт пользователей, покупок и инвентаря с SHA-256.

## Команды

```bash
npm run load:http
npm run load:socket
npm run load:coop
```

Параметры задаются переменными `USERS`, `ROUNDS`, `CONCURRENCY`, `CLIENTS`, `MATCHES` и `BASE_URL`.
