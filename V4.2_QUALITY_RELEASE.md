# V4.2.0 — Rooms, maintenance and Mini App quality release

This release replaces the repeated generic room choices, adds server-authored listening and inspection details, introduces a dedicated maintenance screen, completes building-storage withdrawals, and hardens compact Telegram layouts.

## Gameplay
- 48 generated rooms now use thematic, non-repeating choice sets.
- Existing room definitions are updated during migration instead of being left stale.
- Active expeditions containing the old generic choice trio are safely cancelled and can be restarted.
- Listening and inspection are server-authoritative, limited to three distinct details per room and action, and may reveal a clue or a recommended choice.

## Live operations
- `/api/public-status` is available before Telegram authorization.
- Maintenance mode blocks game APIs and renders a dedicated lift-maintenance screen.
- Title, message, ETA and support URL are editable from the admin panel.

## Building storage
- Players can choose a withdrawal or deposit quantity.
- Transfers use row locking, idempotency and economy audit records.
- Failed idempotent operations can be retried immediately after the underlying problem is fixed.
- The API returns verified inventory and storage balances.

## Interface
- Toast notifications use readable high-contrast text and adaptive duration.
- Buttons, choices, sections, sheets and navigation wrap on narrow Telegram viewports.
- The daily incident ribbon no longer clips its text.
